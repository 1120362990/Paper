## 逻辑漏洞 - 说明一下下面这些场景容易出现什么问题，逻辑

- 用户注册
    1. 恶意用户批量注册
    2. 恶意验证注册账户
    3. 存储型XSS
- 密码找回
    1. 重置任意账户密码
    2. 批量重置用户密码
    3. 新密码劫持
    4. 短信验证码劫持
    5. 用户邮箱劫持
- 订单
    1. 订单信息泄露
    2. 用户信息泄露
    3. 订单遍历
- 登陆
    1. 暴力破解用户名密码
    2. 撞库
    3. 验证码爆破和绕过
        - 某系统登陆是手机验证码，大概1分钟内下发的手机验证码是相同的，即两个手机号能接到相同的验证码，导致任意账户登陆
    4. 账户权限绕过
        - 两个系统账户体系类似，但是是给不同标签的用户登陆的，如果可以互相登陆则为漏洞（某 Src = 2000 RMB）
    5. 手机号撞库
- 后台管理
    1. 用户越权访问
    2. 个人资料信息泄露
    3. 管理员用户名密码绕过
    4. 目录遍历
    5. 字段导入
        - 有个系统支持xls批量导入用户名，xls里面写好XSS语句导入，直接执行了
- 会员系统
    1. 用户越权访问
    2. 个人资料信息泄露
    3. 个人资料遍历
    4. 账户余额盗取
    5. 收货地址遍历、泄露
    6. 非法绑定手机账户和邮箱、篡改
- 传输过程
    1. cookie注入
    2. cookie跨站
    3. cookie劫持
    4. 明文传输
- 评论
    1. POST注入
    2. CSRF
    3. 存储型XSS
    4. 遍历用户名
- 业务查询
    1. 恶意查询
    2. 办理人信息泄露
- 购买支付
    1. 商品金额篡改
    2. 商品数量篡改
    3. 交易信息泄露
    4. 刷单
- 抽奖/活动
    1. 刷取活动积分
    2. 盗刷积分
    3. 抽奖作弊
- 代金券/优惠券
    1. 批量刷取代金券/优惠券
    2. 更改代金券金额
    3. 更改优惠券数量
- 运费
    1. 运费绕过
- 第三方厂家
    1. 盗号
    2. 商家账户遍历
    3. 越权访问其他商家账户
- 充值
    1. 虚假充值金额
    2. 充值数量篡改
    3. 篡改充值账户
